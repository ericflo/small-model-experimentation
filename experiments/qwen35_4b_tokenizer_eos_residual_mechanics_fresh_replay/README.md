materialized first-operation consequences into a residual capability gain over

## Hypothesis

Materializing each candidate first operation's public state-to-target relation
should reduce a depth-three inverse problem to a depth-two residual completion.
If that mechanism is causal rather than surface prompting, it must outperform
name-only and token-preserving shuffled relations, and it must beat ordinary
full-program sampling at both sampled-token and logical-model-token taskwise
first-over budgets. A distinct initial-authorization API and immutable
historical-replay API should remove the parent's instrument failure without
changing any scientific arm, metric, threshold, or selector.
