# Idea intake: transaction-invariant recovery curriculum

## Direction

Install a missing semantic program pattern rather than further optimize loop
syntax: generate diverse executable repairs that require whole-request
validation, copying the complete state, and atomic commit; mix them with the
already successful conditional-recovery task blocks at the action seam.

## Closest prior work

- `qwen35_4b_recovery_verifier_branch_tournament` is the direct predecessor.
  Both local policies retained the tool loop but shared all 20 failures on
  atomic reservations; same-policy sample-more solved only one of 40 stochastic
  trajectories.
- `qwen35_4b_verifier_conditioned_recovery_bank` proved that seven explicitly
  balanced public state→action transitions can install recovery locally when
  plan tokens receive no loss.
- `qwen35_4b_interactive_policy_curriculum` showed that broad live-state traces
  can erase rare verify/commit pivots even when task labels are correct.
- `qwen35_4b_think_ftpo_round2` found true pivot signal but failed context
  locality. Entropy and varentropy helped route/diagnose examples but did not
  identify safe shared-weight token pressure.

## Non-duplication

No prior experiment teaches a general transaction invariant across multiple
fresh procedural repository families while replaying one complete task block
for every conditional loop transition. This is not another interpolation,
policy selector, repetition repair, plan-token update, or success-only trace
compression. The mechanism control replaces transaction tasks with additional
recovery blocks at equal task count and action mass.

## Expected information

- Positive: a concrete missing coding program can be installed locally and
  transfer across API skins while conditional recovery remains intact; this is
  a reusable pattern for failure-core curricula.
- Replay-only positive too: extra recovery dose, not the transaction semantic,
  explains the gain.
- Trained-only positive: the examples teach lexical/API templates, not the
  invariant.
- Locality or retention failure: even action-seam semantic teaching at this
  dose is too broad; reduce or gate the update before any more data mining.
- Full white-box positive but Menagerie null: the invariant transfers within
  procedural coding but is not a current Menagerie bottleneck.

## Decision

Proceed with the fixed primary, matched replay-only control, two independent
family-transfer blocks, broad recovery retention, exact locality, and sealed
Menagerie escalation.
