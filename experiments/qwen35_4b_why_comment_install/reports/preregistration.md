# Preregistration: WHY-Comment Installation (Lifecycle 35)

Frozen before any model event. Lifecycle 35 is cognitive-core coding **bet #4**
and the FIRST of the owner-specified WHY-not-WHAT family. One fresh adapter
trains from BASE in one stage, one merge publishes, and ONE transfer measurement
is read under a frozen, TIGHTENED two-directional consequence. A RETENTION_FAIL
or NULL outcome is a preserved finding, never permission to change this contract
inside this experiment directory.

## The meta-context: three prior signals

- The menagerie aggregate was proven NOT to transfer to real agentic coding
  (McNemar p=1.00 vs base).
- Bet #1 — execution-tracing (`experiments/qwen35_4b_exec_trace_install`), a
  PASSIVE skill — came back NULL: it reshuffled WHICH coding tasks the 4B solves
  without raising the count (HumanEval +1, MBPP -3, agentic 8/35 flat).
- Bet #2 — self-repair (`experiments/qwen35_4b_self_repair_install`), a LOOP
  behavior — was a WEAK POSITIVE (first non-null): HumanEval +3 (76.2->78.0%),
  MBPP -2 (retained), agentic 8/35 -> 10/35 with an ASYMMETRIC discordant (3
  self_repair-only vs 1 base-only — adding, not the flat 5v5 reshuffle of bet #1).
  Underpowered and not individually significant, but directionally positive; the
  lesson recorded was that LOOP/BEHAVIOR curricula beat PASSIVE-SKILL ones.

## The bet: teach WHY, not just WHAT

The owner's next idea is the mechanism most likely to escape `install != convert`:
teach the 4B WHY a correct answer is correct — its causal/generating structure —
not just WHAT the answer is. This cell is the **inline-comment** variant of that
family: the rationale is placed as trailing `#WHY:` code comments bound to the
exact line each explains, leaving the think channel essentially alone.

Each row is a `spec -> correct-solution` episode. A CORRECT small synthetic
function is generated BY CONSTRUCTION from a parameterized family (so we KNOW the
intended algorithm and the role of every line); then EACH meaningful line of the
correct code is annotated with a trailing `#WHY:` comment giving the TRUE CAUSAL
reason that line is correct — the constraint it satisfies, the invariant it
maintains, the bound/edge it handles. The `#WHY:` text is emitted MECHANICALLY by
the family generator (per line, the reason it chose that construct/bound/op), NOT
free-form fabrication and NOT a teacher model.

## Why the inline-comment variant goes FIRST (the cleanest test)

Code comments are INERT to execution grading: the eval runs the code and ignores
comments. So the WHY hypothesis can be tested DIRECTLY, with ZERO annealing —
train the model to write richly-`#WHY:`-commented code, evaluate pass@1 with the
comments ignored. If the CODE got better, the WHY-annotation worked, unconfounded
by any grader that reads the comments. This is why the inline variant is chosen
over the think-block variant (bet #3) to run first.

Crucially, the training prompt is a PLAIN `spec -> function` framing (spec +
signature + tests, matching how HumanEval/MBPP prompt) that does NOT ask for
comments. The WHY-writing behavior must be the model's DEFAULT so it fires on a
plain eval prompt (real transfer), not only when instructed.

## Provenance (no teacher model)

There is NO teacher. The task and its correct solution are built BY CONSTRUCTION
(we build the problem and the solution, so we know why each line is correct), and
every row is confirmed by REAL CPython execution. The `#WHY:` rationale is
generated by the deterministic family generator.

## The no-annealing decision (frozen)

This cell runs NO annealing. The commented model IS the test: because comments
are inert to grading, a raw commented-code SFT is already the clean, unconfounded
probe of the WHY hypothesis. A comment-strip anneal (fine-tune to emit the SAME
code without the comments, then eval) is a documented FUTURE follow-on, executed
ONLY if this cell clears the bar — never inside this cell.

## The frozen, TIGHTENED two-directional consequence (no third state)

Implemented in `measure_transfer.consequence_reading` and unit-tested over its
truth table (including the >=3-problem threshold, the exact 0.02 boundary, and
the retention_fail branch). Identical to bet #2's rule. Let `he_gain =
treat_HE_passed - base_HE_passed` and `mbpp_gain` in PROBLEMS; retention is a
FRACTION tolerance of 0.02.

- **RETENTION_FAIL** (priority) iff EITHER dataset's pass@1 regresses past 0.02.
  A regression past tolerance can never be an install — the forgetting risk
  realized (the longer commented targets could bias generation).
- **INSTALLED_CODING** iff (no regression past tolerance) AND the why_comment
  composite beats base by **>= 3 PROBLEMS** on at least one target dataset
  (HumanEval OR MBPP), with the other retained (>= base - 0.02). Frozen claim:
  teaching WHY inline improved the CODE (inert to the grader); why_comment
  becomes the program reference and funds both the agentic confirm and a
  comment-strip anneal.
- **NULL** otherwise (no >= 3-problem gain and no regression past tolerance).
  Frozen claim: teaching WHY inline reshuffles-without-raising like the
  passive-skill bets; a preserved boundary finding that funds the think-block WHY
  variant (bet #3), not a re-roll of this one.

Also recorded, all DESCRIPTIVE: all four pass@1 numbers (counts + fractions), the
per-problem paired McNemar b/c deltas per dataset, and the per-clause booleans.

## The agentic eval is the PRIMARY real target (manual follow-on)

HumanEval/MBPP are the fast transfer + retention gate. The AGENTIC duet-eval
(base 8/35, self_repair 10/35) is the PRIMARY real target of the program, but is
run MANUALLY as a follow-on confirm on the merged composite (it is not gated in
`measure_transfer`). A verdict here that merely reshuffles the fast signals, with
the agentic eval flat, is honestly a NULL for the mission — the lesson from bet #1.

## Honest priors (computed before any event)

- Precedent FOR: teaching WHY targets the causal/generating structure of a
  correct answer — the most direct attack on `install != convert`; the signal is
  inert to grading, so a positive read is unconfounded; the base already carries
  strong code semantics (HumanEval 76.2%) to build on; the surface is maximally
  divergent from the eval (commented synthetic families vs plain benchmark specs).
- Precedent AGAINST: two of three prior curriculum families reshuffle-without-
  raising, and the one non-null (self-repair) was WEAK and underpowered; a 504-row
  1-epoch dose is deliberately moderate; HumanEval is near a 4B ceiling; a comment
  the model imitates without internalizing the reasoning would not change the
  code; longer commented targets carry a small generation-shift/forgetting risk.
- **P(INSTALLED_CODING, i.e. a >= 3-problem gain with retention) ~= 0.35.**
  Slightly above the self-repair prior (~0.25-0.30) because the WHY mechanism is
  the owner's best theory for escaping install!=convert and the test is cleaner,
  but still below even odds. P(RETENTION_FAIL) ~= 0.10-0.15. P(NULL) ~= 0.50-0.55
  (the modal outcome). Stated plainly: NULL remains a likely verdict, and it is a
  FINDING that prices the inline-WHY mechanism, not a failure.

## Interpretation limits

The verdict prices THIS dose (504 rows, 1 epoch, r32/a64) of THIS curriculum
against base at THIS instrument (greedy pass@1 on HumanEval 164 + MBPP 200). An
INSTALLED_CODING verdict makes the new composite the program reference and funds
the agentic confirm and the comment-strip anneal — it does not itself claim a
confirmed agentic gain. RETENTION_FAIL funds a dose reconsideration. NULL prices
the inline-WHY mechanism and funds the think-block WHY variant. Benchmark firewall
unchanged: execution grades, never data reads.
