# Adversarial design review (workflow) — verdict: sound_with_fixes

## does_it_test_localization
Yes, but only if two framing fixes hold. (a) It genuinely extends C40 ONLY through the LOCALIZATION and REPAIR claims -- pooled AUROC(P vs local-correct) alone is literally C40 run D times and must be demoted from headline. The new result is "confidence has step resolution": argmin-confidence pinpoints the FIRST slip, and targeted repair beats redo-all per unit compute. Foreground those. (b) The error source must be EXECUTE-mode with a stated-but-scrambled order (rule_stated=True, novel/reversal order -> table-lookup difficulty gives a graded per-step error rate). Do NOT use INDUCE-mode for the error-heavy chains: in induce-mode the model is at chance (C39) and, worse, may consistently apply a single WRONG rule -- then local-correctness_i = (m_i == TRUE-rule(m_{i-1})) marks every step wrong even though the model never "slips", so there is no origin to localize and the whole premise collapses. Reusing C37/C40 successor machinery is appropriate; the chain is just s_i = advance(s_{i-1}, k) in the stated order, which is verifiable per-step ground truth (mission-clean, procedurally generated). Add a step-0 anchor (query given) so step 1 is well-defined.

## position_confound
This is the make-or-break issue and same-position comparison + a position baseline is necessary but NOT sufficient. Two confounds are bundled: (a) confidence may decay with step index regardless of correctness (autoregressive drift, growing context, off-manifold state after a slip), and (b) errors cluster late (deeper=harder, propagation). If both hold, argmin-confidence ~ last step ~ where errors are, so localization "works" for the wrong reason. Required disentangling, in order of strength: (1) DE-TRENDED residual confidence -- subtract the per-position mean P (computed over all chains) and localize on the argmin of the RESIDUAL; localization must survive de-trending, else it is pure position. This is the real test and is currently missing from the design. (2) Same-position correct-vs-error contrast: P(correct step | pos i) vs P(error step | pos i), reported WITH per-position n (early positions will be error-sparse -- likely too few to estimate; report the counts and do not hide thin cells). (3) A POSITION-PRIOR localization baseline: pick the step with the highest marginal P(error | position) learned from the data, ignoring confidence; raw-confidence argmin must beat this AND the always-last-step baseline, not just uniform 1/D. (4) Analyze confidence as a function of OFFSET-FROM-FIRST-ERROR (not absolute position): the deployable claim needs the dip to sit AT the origin, but the model may instead dip DOWNSTREAM (confused off-manifold while still locally correct) or double-down and get MORE confident after committing a wrong step. If the minimum is systematically offset from the origin, localization fails even with good pooled AUROC. Also control digit-value (some successor pairs are intrinsically high-P) and scramble-degree (harder orders lower P and raise errors globally -- another reason within-chain de-trending, not pooling, is the right frame).

## local_vs_global
local-correctness_i = (m_i == rule(m_{i-1})), computed against the model's OWN emitted previous step, is the correct ground truth for finding the ORIGIN, and it must be measured with the model's own prefix (the same prefix used to read P(m_i)), not the true chain. Note the FIRST error coincides under both definitions: if steps 1..j-1 are globally correct then m_{j-1}=true s_{j-1}, so local error at j <=> global error at j; local and global diverge only downstream, where local (does step i follow from the model's own m_{i-1}) is the cleaner primary. Handling: (0 local errors) fully-correct chains are excluded from the localization/repair denominators but KEPT for AUROC (all-negative steps) and used for a FALSE-ALARM / wasted-repair metric (does the confident argmin trigger a needless re-try on a clean chain?). (>=1 error) localization denominator = chains with >=1 local error; target = the MIN-index local error. (multiple errors) report the distribution of #local-errors/chain up front as a SANITY GATE: the "single origin, repair it" story only holds if single-slip chains dominate; if error-chains typically have many local slips, the model is not doing clean stepwise reasoning and the repair framing is ill-posed -- pre-register this check and tune difficulty so single-slip chains are the plurality. Report localization two ways: strict (hits the FIRST local error) and loose (hits ANY local error); repair value depends on strict. Also flag coincidental re-correction (slip then land back on true path -- ~10% possible with 10 digits): local-correctness is robust to it; global is not, another reason to keep local primary.

## confidence_read
Reuse C40 exactly but read at each "Step i: " position instead of "Answer: ". Cleanest elicitation: (1) NO-THINK is mandatory -- think triggers code-mode (C39/C40) and, worse, would let the model decide all values in hidden reasoning so the emitted per-step distribution collapses to ~1.0 everywhere (no signal). The "Step i:" scaffold IS the show-your-work channel, so no-think still yields genuine intermediate steps. (2) Use FORCED-SCAFFOLD greedy decoding: force the literal tokens "\nStep {i}: ", let the model emit exactly one token, force the next scaffold, etc. This guarantees D parseable steps, exact position alignment, and one-digit-per-step -- removing the entire format-compliance risk instead of post-hoc parsing. (3) Then read ALL per-step P(m_i) in a SINGLE forward pass over the completed sequence (softmax over the 10 digit ids 15-24 at each "Step i: " position) -- the design's "one forward pass per step" is unnecessarily D passes; one pass total suffices, and it is exactly emission-time P(m_i) because generation was greedy over the forced scaffold. This IS the right signal: P(step i | model's own possibly-wrong earlier steps), which is precisely the deployable read and is correctly matched to local-correctness. MUST pilot for non-degeneracy: confirm P(m_i) is neither ~1.0 everywhere (scaffold-copying) nor ~uniform, and that it varies with local-correctness at fixed position, before scaling n. Also confirm forced-format chain accuracy tracks free-form accuracy (format does not distort competence). For REPAIR the read must switch to sampling (see metrics) -- greedy re-try is a no-op.

## metrics_and_repair
Metrics need three fixes. (1) AUROC: do NOT pool raw across positions (position confound). Report per-position AUROC and a position-controlled/de-trended pooled AUROC; the headline separation must be within-position or on residuals, mirroring C40's within-condition 0.95-vs-0.61 discipline. (2) Localization: argmin-confidence == first-local-error, with baselines uniform-1/D, always-last-step, AND position-prior (argmax P(error|position)); report BOTH raw-confidence and DE-TRENDED-residual argmin -- the residual one is the claim that rules out position. (3) Repair has two real holes. First, GREEDY re-sampling reproduces the same wrong digit -- repair is a no-op unless you re-try with temperature (T>0, best-of-n at that step, majority or P(answer)-verify) or inject "your step j may be wrong, recompute." Specify this; it is currently unspecified and fatal if left greedy. Second, fixing step j REQUIRES recomputing j+1..D because downstream consumed the old m_j. Define the protocol and report both variants: (a) ORACLE-downstream (after re-sampling m_j, recompute the tail with the true rule) = upper bound that isolates "did we find the right origin"; (b) MODEL-downstream (re-run the model from j) = the deployable number. Cost accounting: targeted = 1 resampled step (xN draws) + tail; redo-all = D; report chains-fixed-per-compute so the win is efficiency, not just accuracy. Include the wasted-repair rate on clean chains. n: aim per-step error rate 15-25% via the difficulty mix; with D in 4-6 that gives ~40-60% of chains >=1 error. For localization CI ~+/-0.05 you need ~300-400 error-chains -> ~800-1000 chains total; per-position AUROC needs >=30-50 errors/position, and early positions will be starved -- report per-position counts and expect to tilt toward scrambled orders / more novel chains to populate them. Use multiple k and multiple scrambled orders to avoid single-order idiosyncrasy.

## Confounds
- Position/length: confidence decays with step index independent of correctness (autoregressive drift, off-manifold state) -- only de-trended residual localization rules this out
- Error clustering late: deeper steps harder / propagation, so argmin~last~error for the wrong reason
- Offset-from-error: the confidence minimum may sit DOWNSTREAM of the true origin (confused after slip) or the model may double-down and get MORE confident after a wrong step, missing the origin
- Induce-mode ill-posedness: if error chains come from a consistently-applied WRONG rule, there is no single origin and local-correctness marks all steps wrong -- must use execute-mode scrambled orders
- Multiple-slip chains: if error chains are typically multi-slip, the single-origin repair story collapses; needs a single-slip-dominance sanity gate
- Digit-value: certain successor pairs intrinsically higher-P, correlating with correctness independent of localization
- Scramble-degree/k: harder orders jointly lower P and raise global error -- inflates pooled AUROC without step resolution
- Think-mode / scaffold-copying degeneracy: emitted per-step P collapses to ~1.0 if values were decided in hidden reasoning, killing the signal
- Prefix mismatch: reading P at the TRUE prefix instead of the model's own emitted steps would break the match to local-correctness
- Greedy-repair no-op: re-trying the flagged step greedily returns the same wrong digit, so repair measures nothing unless sampled
- Coincidental re-correction (~10% with 10 digits) makes global-correctness non-monotone; use local as primary

## Must-fix
- Demote pooled per-step AUROC from headline (it is C40xD); make LOCALIZATION + REPAIR the claims that constitute the genuine extension
- Generate error-bearing chains in EXECUTE mode (rule_stated=True) with scrambled/novel stated orders for table-lookup difficulty; forbid induce-mode error chains (no well-defined origin, all-steps-wrong pathology)
- Add DE-TRENDED residual-confidence localization (subtract per-position mean P) as the primary position-controlled test; raw argmin alone is not evidence
- Add a POSITION-PRIOR localization baseline (argmax P(error|position)) and always-last-step baseline; beating uniform 1/D is not enough
- Analyze confidence vs OFFSET-FROM-FIRST-ERROR, not just absolute position, to prove the dip sits AT the origin (not downstream, and not post-slip over-confidence)
- Pre-register a single-slip-dominance sanity gate: report the #local-errors/chain distribution; if multi-slip dominates, the repair framing is ill-posed
- Define localization denominator = chains with >=1 local error; exclude 0-error chains but keep them for AUROC and a wasted-repair/false-alarm rate
- Report localization strict (hits FIRST error) AND loose (hits ANY error)
- Use no-think + FORCED-SCAFFOLD greedy decoding (force "\nStep i: ", emit one digit) to guarantee D aligned parseable steps; read all per-step digit-softmaxes in ONE forward pass over the model's OWN emitted prefix (not the true chain)
- Pilot for signal non-degeneracy: confirm P(m_i) is not ~1.0 everywhere and varies with local-correctness at fixed position; confirm forced-format accuracy ~= free-form accuracy
- Specify the repair mechanism as temperature sampling / best-of-n at the flagged step (greedy re-try is a no-op), and define downstream recompute with BOTH oracle-tail (localization upper bound) and model-tail (deployable) variants, with cost-per-fix accounting vs redo-all
- Size n ~800-1000 chains (mix of D=4-6, multiple k and scrambled orders) targeting 15-25% per-step error rate; report per-position error/correct counts and expect early positions to be starved
