# Adversarial design review (workflow) — verdict: sound_with_fixes

## does_it_test_metacognition
Partly clean, with one structural flaw that must be fixed before it tests self-knowledge at all.

STRENGTH (real): verified ground-truth correctness is genuinely rare in calibration research and it makes this sharper than standard confidence-calibration work. Most calibration studies use MCQA or human-judged correctness where "correct" is defined but the difficulty axis is opaque; here you have a MECHANISTICALLY UNDERSTOOD competence spectrum (execute 1.0 -> familiar-induce 0.45 -> novel-induce 0.12=chance) where you know WHY each cell fails (retrieval works, induction of familiar partly works, induction of novel is impossible). That lets you ask not just "is confidence calibrated" but "does confidence track the KNOWN retrieval/induction boundary." Good.

THE FLAW: the crisp headline ("on novel-induce, is it confident or uncertain?") lives in a BETWEEN-condition comparison (mean confidence on novel-induce vs execute), and between-condition confidence is NOT a test of self-knowledge — it is confounded by surface features that co-vary perfectly with condition (see surface_cue_confound). A dumb regex keying on "is there a scrambled-order block" achieves near-perfect between-condition AUROC while knowing nothing about the model's own computation.

The ONLY surface-matched test is WITHIN-condition item-level calibration, and there the substrate fights you: (a) execute is ~1.0 so there are essentially NO negatives -> within-condition AUROC is undefined; it only anchors the top of the scale. (b) novel-induce is AT CHANCE (0.12 vs 0.10), so per-item correctness is pure luck — there is no stable easy-vs-hard structure to detect, and within-condition AUROC is ~0.5 BY CONSTRUCTION regardless of whether the model has self-knowledge; item-level metacognition is essentially undefined at chance. (c) That leaves familiar-induce (~0.45) as the ONE cell where within-condition item-level calibration is both defined and surface-matched — intermediate accuracy with real item-to-item difficulty variance (k, which digits were seen, gap from query to nearest seen item). So the actual self-knowledge test must be relocated to familiar-induce item-level calibration; the novel-induce headline is a mean/abstain-rate question, not an AUROC question, and must be labeled as the confounded (deployment-heuristic) measure, not privileged introspection.

## surface_cue_confound
This is the crux and within-condition + k-variation is NECESSARY but NOT SUFFICIENT.

Why within-condition helps: within novel-induce every item carries the same scrambled-order block, so any confidence variance there cannot come from block-presence — surface is matched. Correct in principle. But (per above) novel-induce is at chance so there is no correctness signal to calibrate against; and familiar-induce has NO scrambled block, so its within-condition variance is driven by k/seen-set/query-gap — that is genuine ITEM-level metacognition, but on a DIFFERENT axis than the scrambled cue. So within-condition calibration validates self-knowledge in general but does NOT directly test the scrambled-cue confound.

k-variation: varying k in INDUCE mode changes difficulty without a surface label (k is latent in few-shot), which does decouple difficulty from a visible marker — good, but you must first EMPIRICALLY verify k actually grades difficulty (it may not; k=5 "halfway" etc.). Report accuracy-by-k before trusting it.

What is additionally required:
1. FORMAT-EQUALIZE across conditions: always render an order block, stating the natural order "0 1 2 ... 9" for familiar too, so block-presence/absence can no longer discriminate conditions. Then familiar vs novel differ only in block CONTENT (natural vs scrambled), not format.
2. DISSOCIATION condition (the decisive one): a scrambled-LOOKING order that is actually easy/regular — e.g. a reversal (9 8 7 ...) or a simple relabeling the model CAN induce. If confidence drops on scrambled-looking-but-easy as much as on genuinely-hard scrambled, confidence is a surface heuristic. If confidence stays high exactly where the model actually succeeds despite the scary-looking block, that is competence-assessment beyond surface. This is the clean surface-vs-competence contrast the current 3 conditions cannot make.
3. INSPECT THE MODAL WRONG ANSWER on novel-induce: if the model systematically applies the NATURAL-order successor (ignoring the stated scrambled order), failures are a consistent wrong-rule, and confidence-drop-from-seeing-scramble is surface, not knowledge that its own answer is wrong.
4. EXTERNAL SURFACE BASELINE (see metrics): the formal disentangler — self-confidence must beat a classifier built only from prompt-surface features at WITHIN-condition per-item prediction. Genuine tasks-it-fails-for-non-obvious-reasons live inside familiar-induce (items it gets wrong under a friendly natural-order surface); confidence flagging THOSE is the strongest evidence of true competence-assessment.

## elicitation_and_corruption
(1) SAME-PASS DOES NOT CORRUPT THE ANSWER if you order it answer-first: emit `Answer: <digit>` THEN `Confidence:`. Autoregressively the digit token is committed before any confidence token exists, so confidence cannot change it. A separate confidence pass is WORSE — it forces the model to re-read and rationalize a shown answer. Keep it single-pass, answer-first. (Does asking for confidence perturb the reasoning that precedes the answer? Negligible with answer-first tail placement; not worth a separate pass.)

(2) CONSTANT-CONFIDENCE FLOOR is a real and likely failure for a 4B with an RLHF "sound confident" prior. Detect it explicitly: report the DISTRIBUTION of verbalized confidence (SD, and fraction equal to the modal value); if SD~0 or >~80% of values are one number (commonly 90/100), the explicit channel is degenerate and mean-confidence-by-condition is meaningless. Do NOT rely on the raw 0-100. Fallbacks, which should be PRIMARY not backup: (a) the answer-token probability itself — since the output is a single digit, read P(chosen digit) directly from one forward pass; (b) P(True) second-pass (Kadavath-style): "Is the proposed answer correct? True/False" and take the True-token probability — continuous and non-degenerate even when verbalized numbers flatten; (c) a discrete Likert (unsure/even/likely/certain) which small models populate more variably than a 0-100 field.

(3) NO-THINK CAVEAT (important): C39 correctly used no-think to avoid code-mode for the ANSWER, but no-think gives the model almost no room to introspect, which biases explicit confidence toward flat/overconfident and will UNDER-measure metacognition. Run BOTH no-think and think for the confidence elicitation (the single-value task is execution-safe; keep the codemode_rate monitor). Do not assume no-think is right for the confidence channel just because it was right for the answer channel.

(4) SCALE: verbalized 0-100 is fine only if non-degenerate and must be bucketed for ECE. For a 4B the honest per-item signal is the answer-digit probability / margin over the 10 digit tokens (implicit) rather than a self-reported number. Prefer binary-will-I-be-right / P(True) over free 0-100 as the explicit measure.

(5) ABSTENTION is under-elicited by default: a helpfulness-tuned model rarely abstains, so abstain-rate will be ~0 everywhere and useless. Give an explicit scoring rule in the prompt (e.g. +1 correct, -3 wrong, 0 abstain) to make abstaining rational. Note this makes abstention a decision-theoretic response to the payoff, not pure self-knowledge — but WHICH items it abstains on is still the signal; report abstain item-level AUROC. Also clean answer extraction (constrained single-token decode, not the regex last-digit fallback) because extraction errors mislabel correctness and pollute every calibration metric.

## metrics_and_baselines
METRICS — mostly right, with fixes:
- Correctness / mean-confidence / abstain-rate by condition: keep, but label between-condition confidence a DEPLOYMENT HEURISTIC (surface-confounded), not self-knowledge.
- AUROC(confidence vs correctness): compute WITHIN condition and report it as the headline; pooled cross-condition AUROC is the surface artifact (near-guaranteed positive, meaningless) — report it only to show it is inflated relative to within-condition. Execute (no negatives) and novel-induce (all-luck) cannot support within-condition AUROC; state this and do NOT report it there. Familiar-induce is the load-bearing cell.
- ECE + reliability diagram (binned); with degenerate confidence ECE collapses to one bin — report the confidence histogram alongside.
- Selective-prediction (accuracy vs coverage): meaningful pooled and within familiar-induce; at chance-level novel-induce there is nothing to select on (all items equal). Bootstrap CIs on every curve.

EXTERNAL SURFACE BASELINE (this is the decisive comparator, C30/C31 pattern) — define concretely: a simple classifier (logistic reg / small tree) predicting per-item correctness from PROMPT-SURFACE features ONLY, no model self-report: {condition, order-block-present, k-if-stated, #examples, query digit, gap from query to nearest seen input, is-order-scrambled}. The self-knowledge claim requires the model's own confidence to BEAT this classifier at WITHIN-condition (familiar-induce) per-item prediction — because within condition the surface features are largely matched, so beating the classifier means the model reports something about its own computation the surface does not reveal. If self-confidence AUROC <= surface-classifier AUROC, it is not privileged self-knowledge, just surface-readable difficulty. Apply the identical test to sample-entropy. Also include the trivial always-confident baseline for ECE reference.

MIN-N: the binding constraint is positives AND negatives per class WITHIN familiar-induce. At 0.45 accuracy you want >=~40-50 per class for a stable AUROC/ECE, so n>=~200 familiar-induce items (n=100 is borderline; n=60 is too few). Execute: no errors -> undefined, skip within-AUROC. Novel-induce at n=60 gives ~7 positives (and they are luck) -> AUROC CI is enormous and uninterpretable; either raise to n>=300 if you insist on reporting it, or (better) report only mean-confidence/abstain there. Add multiple task-seeds and multiple k values, bootstrap CIs on AUROC/ECE. Between-condition mean comparisons are fine at n=60.

## implicit_vs_explicit
Worthwhile and genuinely the most interesting axis — an implicit-knows / explicit-doesn't dissociation would itself be a finding (the model has distributional uncertainty it cannot verbalize). Keep it, with three fixes.

1. USE THE DIRECT TOKEN DISTRIBUTION, not Monte-Carlo sampling. The output is a SINGLE digit, so one forward pass gives the exact next-token distribution over the 10 digit tokens -> exact entropy, exact P(answer), exact top-2 margin, with zero sampling noise. This strictly dominates k-sample entropy for a single-token output. Constrain the format so the answer digit is one token at a known position (e.g. force "Answer: " then one digit). Use MC sampling only if you keep the free prose-then-digit format where the digit position varies; if so, k=20 samples (10 is too few to estimate a 10-way entropy), temperature T=1.0 to read the model's NATIVE distribution (low T collapses the signal to greedy; high T adds sampling noise unrelated to belief) — report T=0.7 and 1.0. Score agreement as the empirical entropy / top-1 vote share over the 10-way digit output.

2. GUARD ENTROPY-vs-DIFFICULTY with the SAME external baseline. Sample-entropy is high on hard items and low on easy ones, so "entropy predicts correctness" is partly "difficulty predicts correctness." It counts as self-knowledge only if entropy beats the surface-feature classifier at WITHIN-condition per-item prediction. If it does not, it is just difficulty, not introspection.

3. THE KILLER CONFOUND for implicit at chance — CONFIDENTLY WRONG. If on novel-induce the model applies a systematic wrong rule (most likely the NATURAL-order successor, ignoring the stated scrambled order), its samples will be LOW entropy (consistent) but WRONG. That is the worst silent-failure case and it makes implicit confidence ANTI-calibrated on exactly the condition the headline cares about. So the mechanism decides the result: novel-induce failure that is high-entropy scatter = self-aware; low-entropy consistent-wrong-rule = silent. Pre-register inspection of the modal wrong answer and the entropy distribution on novel-induce. This is more informative than a single implicit-vs-explicit AUROC number.

Net: implicit (direct token entropy/margin) is likely the stronger and more honest signal for a 4B than verbalized confidence, but both must clear the external-surface-baseline bar, and the confident-wrong mode must be checked or the implicit story can be exactly backwards.

## Confounds
- Between-condition confidence is surface-confounded: order-block-present and stated-vs-fewshot co-vary perfectly with condition, so a regex achieves the 'headline' AUROC without any self-knowledge; the crisp headline lives in the confounded measure.
- Novel-induce sits at chance (0.12 vs 0.10) -> per-item correctness is luck with no learnable easy/hard structure, so within-condition item-level AUROC is ~0.5 by construction and metacognition is essentially undefined there.
- Execute is ~1.0 -> essentially no negatives, so within-condition calibration is undefined; it only anchors the top of the confidence scale.
- Verbalized 0-100 from a 4B is likely degenerate/constant (RLHF overconfidence prior), flooring the explicit channel and making mean-confidence-by-condition meaningless.
- Prompt-FORMAT cue: familiar/execute lack the scrambled-order block that novel carries, so block-presence alone predicts condition and can drive confidence without competence-introspection.
- Systematic-wrong-rule risk: on novel-induce the model may consistently apply the natural-order successor -> low-entropy CONFIDENT WRONG, making implicit confidence anti-calibrated exactly where the headline is; must inspect the modal error.
- Answer-extraction regex (last-digit fallback) mislabels correctness, polluting every calibration metric that depends on the correctness label.
- no-think (needed to avoid code-mode for the answer) gives no room to introspect and biases explicit confidence toward flat/overconfident, under-measuring metacognition.
- Sample-entropy conflates task-difficulty with self-knowledge unless it beats an external surface-feature classifier within condition.
- Abstention is under-elicited without a scoring rule: a helpfulness-tuned model abstains ~0% everywhere, making abstain-rate uninformative.
- MC sample-entropy on a single-digit output adds needless noise and depends on temperature; low T collapses the signal, and the exact token distribution is available in one forward pass.

## Must-fix
- Relocate the self-knowledge HEADLINE to WITHIN-condition item-level calibration in the intermediate-accuracy cell (familiar-induce ~0.45), where surface is matched and error variance is real. Report between-condition mean-confidence/abstain separately and explicitly label it a surface-driven deployment heuristic, NOT privileged self-knowledge. Do not report within-condition AUROC for execute (no errors) or novel-induce (all-luck).
- Add the EXTERNAL SURFACE-FEATURE baseline as the decisive comparator: a classifier from prompt-surface features only {condition, block-present, k, #examples, query, gap-to-nearest-seen, is-scrambled}. Self-confidence (and sample-entropy) count as self-knowledge only if they BEAT this classifier at within-condition (familiar-induce) per-item prediction.
- Format-equalize all conditions (always show an order block, stating natural order for familiar too) so block-presence cannot be the cue; ADD a dissociation condition = scrambled-LOOKING but actually-easy order (reversal/regular relabel). Confidence dropping there as much as on truly-hard scrambled = surface heuristic, not competence.
- Use the DIRECT next-token distribution over the 10 digit tokens (one forward pass) for exact entropy/P(answer)/margin instead of MC sample-entropy; constrain output so the answer digit is a single known-position token, and clean the correctness label with constrained decoding (drop the regex last-digit fallback).
- Add logprob-based explicit signals as PRIMARY (P(True) second pass; answer-token probability) since verbalized 0-100 will likely be degenerate; pre-register a degeneracy check (confidence SD and modal-value fraction) and report the confidence histogram, not just the mean.
- Fix n and CIs: >=~200 familiar-induce items for a stable within-condition AUROC/ECE (balanced classes), multiple seeds and k values, bootstrap CIs; explicitly note execute and novel-induce cannot support within-condition AUROC.
- Inspect the MODAL WRONG answer and entropy distribution on novel-induce to distinguish high-entropy scatter (self-aware) from low-entropy consistent-wrong-rule (silent failure, e.g. applying the natural-order successor) — the mechanism, not a single AUROC, decides the headline.
- Elicit answer FIRST then confidence in one pass (autoregressive -> answer uncorrupted); run BOTH no-think and think variants (single-value task is execution-safe; keep the codemode monitor) so explicit metacognition is not under-measured; give a proper scoring rule to make abstention rational and report abstain item-level AUROC.
