# Adversarial design review (workflow) — verdict: sound_with_fixes

## token_matching
The A (prompt->code) and T (prompt->thinking->code) sequences differ almost entirely in the thinking span (median ~2500 chars/trace per C27), so at equal epochs T processes far more tokens and takes more/longer gradient steps. Key distinction: the CODE tokens (the shared supervision target) are identical per example between A and T; the extra is supervision on thinking tokens, which is INTRINSIC to the treatment -- you cannot strip it without deleting T. So "matching" means ruling out that T merely got more optimizer work, not equalizing sequence length. Concrete plan: (a) epoch-matched A vs T (same #passes over the same tasks) as the natural same-data-budget primary; (b) a step/token-matched A that trains extra epochs until total optimizer steps AND tokens processed approx equal T's, to show T's win is not just more gradient. Report BOTH; robust only if T>A under epoch-match AND step-match. Add a loss-MASK ablation (T-masked: thinking present in context via teacher forcing, loss computed on code tokens only) to separate 'supervision on reasoning' from 'reasoning merely in context'. Critical guard: token-matched A via repeated epochs just memorizes (diminishing returns), so a T>A result is meaningless unless A is shown COMPUTE-SATURATED -- report A's epoch dose-response (val loss + greedy@1 vs epochs) proving more epochs stop helping. Otherwise T>A is indistinguishable from 'A undertrained'.

## thought_source_distillation
It is mission-compliant self-training, NOT an external teacher: banked_1280 is the same frozen Qwen3.5-4B base plus a self-trained LoRA whose code capability was produced by interpreter brute-search (C24); no other model's weights ever enter the pipeline and the interpreter is only a verifier. Harvesting its think traces to train a fresh gen-N+1 adapter is exactly the M4 expert-iteration/STaR flywheel. BUT it is NOT base self-bootstrapping: base think-coverage ~0, so the traces are not base-reachable and T is distilling banked_1280. That is fine for the A-vs-T headline BECAUSE both A and T distill banked_1280 on identical {prompt,code} targets, so the only delta is the reasoning trace and T>A isolates the reasoning channel rather than smuggled code capability -- PROVIDED you verify A's and T's code targets are byte-identical and both are trained fresh-from-base. Honesty framing: call it 'self-distillation across a self-trained generation,' print base-think cov~0 right next to it, and never claim the 4B reasoned to new capability from scratch. Cleanest keep-honest controls: (1) T-continue = continue FROM banked_1280 on the traces; if T-continue approx banked_1280 the traces add nothing beyond the weights that produced them (pure rationalization), if T-continue > banked_1280 on step-1 the reasoning channel genuinely adds. (2) Trace-source dose: also harvest from a weaker adapter (banked_160); if T-quality tracks the SOURCE adapter's coverage, the capability is inherited from the source, quantifying teacher-share vs reasoning-channel-share. Does it matter for the claim: yes for WORDING -- the defensible claim is 'banking the reasoning narration installs more usable planning than banking answers alone, within the same self-trained source,' not 'reasoning creates capability.' Keep fresh-from-base T-vs-A as headline, continue-from-banked as the discriminator.

## rationalization_risk
Real and central. banked_1280 already knows the depth-3 code (its LoRA), so its think traces are plausibly answer-first rationalizations rather than forward search -- C26 explicitly observed 'confused meta-reasoning, not enumerate-and-test' in this exact setting. Verification keeps only code-correct traces, but code-correctness does NOT validate the reasoning: a ritualized/wrong rationalization can sit atop correct code (selection bias toward answer-first traces). If so, T installs recognition/retrieval-with-narration, not planning. Detect/handle: (1) PRE-REGISTER the C25/C26 STEP-1 next-op ranking (goal 3 ops away, no intermediate state materialized) as the PRIMARY, rationalization-robust metric -- recognition inflates step-3 and greedy@1, but only genuine forward planning lifts step-1, which is precisely where C26/C27 test-time thinking failed. A durable step-1 lift is the clean positive; a deployability/step-3 win with a flat step-1 = recognition, not planning. (2) Training-side content-causality ablations: T_corrupt (fluent-but-WRONG/shuffled thinking, same code) and T_filler (generic filler thinking); if T approx T_corrupt the thinking is inert ritual. (3) Test-side: force-think then truncate/replace T's own thinking and check whether code degrades (does its generated reasoning causally drive its code?). (4) Annotate a trace sample for genuine search structure and report the fraction; check the thinking does not merely restate the code (answer-leakage into thinking = T is a copier). Does it invalidate the headline: it SPLITS it. 'T>A on deployability' can survive as a real usable-capability lever even if traces are rationalizations; but the stronger 'banking reasoning installs PLANNING' claim requires the step-1 lift AND content-causality to hold. Report both channels; do not relabel a deployability/recognition win as planning.

## deployability_metric
T is think-trained, A no-think-trained, so the fair headline is each recipe in its BEST usable config: T-think greedy@1 vs A-no-think greedy@1 vs base -- 'most usable planning per recipe.' But run the FULL grid {base, A, T} x {no-think, think} for greedy@1 + coverage@k, ALL in-run on one harness (do NOT inherit base cells as C27 did). The off-diagonal cells are load-bearing: T-no-think isolates whether reasoning-banking improved the WEIGHTS (helps even without a test-time thinking channel) vs merely unlocking a test-time channel; A-think is the C24 sample-more/think baseline (does A gain from test-time thinking despite no-think training). Confound to kill: T-think spends more INFERENCE tokens than A-no-think, so a raw win conflates capability with test compute. Fixes: report generated-token counts per cell; add a budget-matched comparison (cap T's thinking, compare at equal total tokens); and it MUST beat the mission's sample-more baseline (A no-think coverage@k with k set to match T-think's token budget). Headline = T-think greedy@1 minus A-no-think greedy@1 WITH test-compute reported, PLUS the parse-immune step-1 ranking as the planning-specific headline (greedy@1 for T-think needs post-</think> code parsing, which is fragile; the ranking channel avoids it). Avoid confounding trained-mode with capability by ALWAYS reporting the off-diagonal cells so both 'trained-mode-best' and 'matched-mode' comparisons are visible.

## Confounds
- Code-capability smuggling: T inherits banked_1280's code unless A's and T's {prompt,code} targets are byte-identical -- verify the only diff is the thinking span and report it
- Training compute/token budget: T's longer sequences give more gradient work; needs step/token-matched A PLUS an A epoch dose-response proving A is compute-saturated (else T>A = A undertrained)
- Test-time compute: T-think generates more inference tokens than A-no-think; needs token-budget-matched comparison and a sample-more (A no-think coverage@k at matched budget) baseline it must beat
- Trace selection validity: verification is on CODE not reasoning, so correct code can carry rationalized/answer-first thinking (selection bias)
- Held-out leakage: frozen n=80 must be 0-leak by function-signature AND op-composition vs BOTH banked_1280's training set AND the new harvest pool
- Answer-leakage into thinking: banked traces may restate the code, making T a trivial copier rather than a planner
- Statistical power / single seed: step-1 deltas (~0.025->0.175) sit near C26/C27 noise at n=40; use n=80, >=2 training seeds per arm, pre-registered primary + Wilson CIs
- Inherited vs in-run base cells: C27 inherited base from C26 -- rerun the full grid in one harness
- Source-adapter capability leakage: T may track banked_1280's coverage rather than the reasoning channel -- trace-source dose control (banked_160 vs 1280) quantifies it
