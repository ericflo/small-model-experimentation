# Adversarial design review (workflow) — verdict: sound_with_fixes

## apples_to_apples
Yes, as specified it is apples-to-oranges, on at least four axes, and the direction of the bias is not obviously in favor of "no lift" — it can manufacture a spurious lift OR mask a real one, so it's just uninterpretable. (a) SAMPLE SPACE: C25's 0.013/0.062/0.237 floor is a 32-way CLOSED-SET forced argmax over exact op-strings (score_ops, length-normalized logprob, one forward). Generation is OPEN: it can emit a malformed op, wrong format, refuse, or truncate — every such event scores wrong, so generation "chance" ≈ 0, not 1/32. Comparing a generated-top-1 number against a ranking floor of 0.013 or a ranking chance of 0.031 is comparing numbers from different sample spaces. (b) DECODE RULE: ranking picks argmax of length-NORMALIZED sequence logprob; greedy generation picks token-by-token argmax — these are not the same op even at B=0. (c) The C25 log explicitly says the generate-and-parse channel was ABANDONED because the model "reasons verbosely and gets truncated," which is why they switched to logit ranking. Phase A B=0 revives precisely the degenerate channel C25 discarded, so "no-think generation" is a strictly worse floor than C25's 0.013, and any B>0 "lift" over it may be lift over a self-inflicted-low baseline, not over the real forward-pass capability. (d) PARAM SCORING: ranking enumerated all 32 op/param combos as exact strings; generation must PRODUCE add_k(2) with the right param in prose — asymmetric difficulty and asymmetric canonicalization. FIX: (1) Do not compare think-generation to C25's ranking floor at all. (2) Run a no-think GENERATION baseline (B=0) in the identical prompt+parse+canonicalization harness, forced-answer (re-prompt/constrain until a well-formed op is emitted, or count parse-fail explicitly). That is the fair budget-0 anchor. (3) ALSO run think→RANK: think for B tokens, close </think>, then run the SAME 32-way likelihood ranking. This holds the channel fixed at ranking and isolates budget, and it is directly commensurable with C25's 0.013. The headline should be think-rank vs no-think-rank (channel = ranking) AND think-gen vs no-think-gen (channel = generation) — two within-channel curves. If the effect only appears across channels (think-gen vs no-think-rank), it is a channel artifact, not a compute effect. Score think by BOTH generation-top-1 and ranking-after-thinking; a genuine planning lift should show in the ranking-after-thinking number, which is immune to parse/truncation noise.

## budget_matching
The honest axis is model compute (generated+forward tokens, or FLOPs, or wall-clock) — NOT interpreter calls — and on that axis the "beats brute" framing is at risk of meaning "spent 100-1000x more model compute." C25 matched on ~1088 interp-calls/task. But brute-force pays ~0 model cost (it enumerates 32 ops, no thinking); think-guided search pays B tokens of GENERATION per expanded node (hundreds to thousands), plus the forward pass. So matching on interp-calls hides the entire resource the claim spends. On a model-token/FLOPs/wall-clock axis, brute-force is effectively free and think-guided is enormously expensive, so the comparison can invert: brute wins on the axis that actually costs. MUST report at least two axes explicitly: (i) interpreter calls (where guidance can look efficient) and (ii) total model tokens generated end-to-end (where brute ≈ 0). State which axis the claim lives on. The ONLY clean, non-compute-inflated win is a COVERAGE-CEILING win: think-guided solves true-depth-3 tasks that brute-force cannot solve at ANY budget (i.e. brute's asymptote < think's asymptote). C25 already found the opposite shape for banked guidance — competitive-not-superior, brute high-budget 0.81 > banked 0.225 — and the sibling frontier experiment found decompose guidance buys "efficiency, not coverage; brute matches/beats at high budget." If Phase B reproduces efficiency-on-interp-calls-but-plateaus-below-brute, that is NOT breaching the wall; it is re-deriving C25/frontier. Also: "monotone in B" for Phase A is fine as a step-accuracy curve, but for Phase B a fair fixed-total-token budget must be split across nodes — 2048 tokens/node means far fewer nodes explored than brute at equal wall-clock, so the beam depth/width and node count must be reported alongside B, not just interp-calls.

## internal_bruteforce
This is the central threat and it DOES matter, because it changes what the result means. With only 16 ops / 32 combos, the model can literally enumerate candidates in its scratchpad, mentally apply each to the current list, and compare to the goal — i.e. run the interpreter inside the CoT. If that is the mechanism, a step-1 lift is NOT "latent planning/lookahead surfacing"; it is "the model can emulate the interpreter in tokens," which reframes the headline from "depth is latent, breach the forward-pass wall" to "serial test-time tokens let the model be its own interpreter." For the weak claim (serial test-time compute breaches the wall training-free) internal simulation still counts as a real training-free elicitation. But for the STATED claim (a lookahead/PLANNING limit that serial compute breaches, "depth is latent"), it does not — and it makes Phase B double-counting: external-brute vs (external-brute-moved-into-tokens + interpreter), with the internal brute uncounted on the interp-call axis (ties back to the budget point). Probes to separate: (a) READ THE TRACES — classify CoT as enumerate-and-test ("try sort_asc: [..]; no; try add_k..") vs goal-directed reasoning; report the fraction. (b) VARY DSL SIZE (16 → 64 → 128 ops, branching factor up): genuine planning should degrade gracefully; internal enumeration should collapse once the branching factor exceeds what B tokens can simulate, and step-1 accuracy should track roughly (B/cost-per-candidate)/branching — if it does, it's enumeration. (c) ABLATE/CORRUPT THE GOAL: give a wrong or hidden target-list; enumeration-against-goal should break in a specific way planning-from-structure would not. (d) Check whether think-gen first-op accuracy is bounded by feasible-candidates-simulated-within-B. If the DSL-size sweep shows collapse and traces show enumeration, report the result honestly as interpreter-emulation, not planning.

## what_success_shows
Be careful: the most likely "lift" is nearly vacuous, and the genuinely informative cell is narrow. The RANK phase materializes the TRUE intermediate state before steps 2 and 3 (rank_phase applies the ground-truth ops to advance `states`), so at step 2 the model is HANDED the correct depth-2 list and only has to recognize a depth-≤2 remainder — capability C12/C25 already showed it has (step-3 recognition 0.237). Therefore a lift at steps 2/3 shows decomposition + external state-materialization, i.e. the interpreter feeding the model, NOT latent lookahead. STEP 1 is the only clean test of the wall: current==input, goal 3 ops away, no intermediate help, base=0.013 (below chance 0.031). So the headline must foreground STEP-1, and a step-1 lift with NO intermediate states is the only result that legitimately shows latent lookahead being elicited. Given C23 base monolithic-think depth-3 ≈ 0.00 (free-form code) and the new monolithic-op-sequence control: (i) if step-wise (2/3) lifts but step-1 and monolithic-op-sequence stay ~0, the win is decomposition+state-materialization, and it reconciles with C23 without touching "lookahead is latent." (ii) If monolithic-op-sequence lifts above C23's 0, that's a FORMAT effect (emit DSL not Python), orthogonal to lookahead. (iii) Only a step-1 lift, robust in the ranking-after-thinking channel and not explained by internal enumeration (see DSL-size probe), supports "the wall is a single-forward-pass limit breached by serial compute." Conversely, NO lift is NOT automatically a clean refutation: the abandoned-generation channel truncates verbose reasoning at B tokens, so a flat curve could be parse/truncation artifact, or B=2048 could be below the needed reasoning length. A no-lift is only a refutation if (a) parse-success is high, (b) the ranking-after-thinking channel (parse-immune) is ALSO flat, and (c) the curve is flat/decreasing rather than still-rising at B=2048. Report parse-fail and truncation rate per budget, and both parse-conditioned and unconditioned accuracy.

## Confounds
- Channel/sample-space confound: C25 floor is closed-set 32-way length-normalized ranking (chance 1/32); Phase A think is open generation (chance ~0, parse-fail=wrong). Comparing them, or comparing think-gen to chance 0.031, is invalid.
- State-materialization confound: rank_phase hands the model the TRUE intermediate list before steps 2/3, so 2/3 test depth-<=2 recognition (already established), not lookahead. Only step 1 (3 ops away, no help) tests the wall.
- Parse/truncation confound: the C25 log records generate-and-parse was abandoned for verbose-truncation; B=512 may be below reasoning length, so a flat curve can be truncation artifact, not absence of planning. Must report parse-fail/truncation rate and a parse-immune ranking-after-thinking channel.
- Budget-axis confound: matching on interpreter calls hides B-tokens x nodes of model compute that brute-force never pays; on a model-token/FLOPs axis brute is ~free and the 'beats brute' can invert.
- Internal-brute-force-simulation confound: with 16 ops the CoT can emulate the interpreter; a step-1 lift may be enumeration-in-tokens, not planning. Needs DSL-size sweep + trace inspection to separate.
- Decode-rule confound: ranking uses length-normalized sequence logprob; greedy generation is token-wise argmax; even at B=0 these pick different ops.
- Param-scoring/canonicalization asymmetry: ranking enumerated exact op(param) strings; generation must emit the param in prose. Scoring must use identical canonicalization or it biases the comparison.
- Single-draw/stochasticity confound: think at B>0 is sampled; a single greedy or single-sample generated top-1 vs a deterministic 32-way argmax is noisy. Need multiple samples/seeds and reported variance; C25 tested 'one beam' for Phase B.
- Task-population confound: sibling frontier retro-audit found 40% of nominal depth-3 tasks are behaviorally depth<=2. Confirm this experiment reuses C25's 80 min-depth-VERIFIED true-depth-3 held-out set, not a fresh unaudited draw.
- Monolithic-control interpretation confound: if monolithic-op-sequence lifts over C23's free-form-code 0, that's a format effect (DSL vs Python), which must not be read as evidence about lookahead.

## How this experiment RESPONDED
PRIMARY metric is think->RANK vs no-think->RANK (channel-matched to C25's likelihood ranking, immune to parse/truncation), NOT think-generation vs no-think-ranking. HEADLINE is STEP-1 (goal 3 ops away, no intermediate state materialized -- the only clean lookahead test; steps 2/3 are contaminated because the harness feeds the true intermediate state, reducing them to depth-<=2 recognition the model already has). Internal-brute-force threat addressed by (a) dumping + classifying step-1 thinking traces (enumerate-and-test vs goal-directed) and (b) a DSL-size sweep (does step-1 lift collapse as branching grows?). Budget reported on the model-TOKEN axis (brute ~0); the only clean 'breach' is a coverage-ceiling win (solve tasks brute can't at any budget). Weak vs strong claim separated: internal simulation still = real training-free elicitation (weak claim), but not latent PLANNING (strong claim).
