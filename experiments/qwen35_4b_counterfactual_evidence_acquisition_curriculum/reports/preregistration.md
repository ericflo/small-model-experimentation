# Preregistration: counterfactual evidence-acquisition curriculum

This design is frozen in content before any model-bearing call. The immutable
design commit, file digests, and exact parent identities must be written to a
design-lock receipt before GPU generation or training. A rebase may update the
commit identity only if every frozen-file digest remains byte-identical and the
old and new commit IDs are recorded.

## Claim under test

Action-only supervision balanced at conditional state-to-action transitions
can install a deployable coding policy that searches discriminating public
repository evidence before its first patch and binds that evidence to the
correct semantic proposal. The policy must transfer to unseen task families,
evidence paths, and query wording; beat the exact start checkpoint, the C54
incumbent, aligned-dose controls, and actual-compute-matched sampling; and
retain locality plus the full rejected/failed/verify/commit loop.

The mechanism is falsified if aligned training does not beat a within-dyad
label shuffle and an explicit-contract equal-dose control, if a deterministic
matched-operator search with nondiscriminating output closes the same
qualification gap as the correct search, or if any apparent trained-family gain
fails the frozen transfer or retention gates.

## Firewall and hidden-label boundary

All scientific tasks are fresh procedural Python repositories. The model may
see issue text, public files, the file tree, public search/read output, visible
test output, and ordinary tool errors. Hidden executable code, hidden results,
counterfactual branch labels, oracle patches, and host workspace scores never
enter a message or adaptive model decision.

Hidden checks may score a changed workspace after generation, but their result
does not alter the conversation, stopping rule, sample-prefix membership, or
candidate selection. No file under `benchmarks/` may be read or imported.
Menagerie remains sealed and may later be invoked only through its public CLI,
with aggregate receipts.

Each aggregate event requires a pre-call seed reservation and is authenticated
by its canonical event digest, exact white-box authorization and design-lock
hashes, live full-checkpoint fingerprints, frozen bench/analyzer hashes, and the
public Menagerie Git-tree identity. A preseeded or stale aggregate log cannot be
resumed.

Every registered scientific stop must emit a terminal-disposition receipt.
That receipt remains explicitly open until the README, report, log, program
evidence/backlog, synthesis/claim decision, brief/chart, catalog, and artifact
manifest are committed and pushed to `main`; a second closeout seal binds that
pushed documentation commit. The seal must itself be cataloged, committed, and
pushed, after which a non-mutating verifier requires the receipt and generated
indexes to be clean at `origin/main` and reruns `make check`. A stopped run is
not lifecycle-complete merely because its gate or local closeout receipt exists.

## Counterfactual substrate

The acquisition training families are `spec_lookup`, `spec_tags`,
`spec_duplicates`, and `spec_score`. Held-out transfer families are
`spec_switch` and `spec_update`.

Each inferred dyad contains two repositories with:

1. byte-identical issue text, source, tree and path names, and every
   non-discriminating public file;
2. exactly one designated public evidence file whose bytes differ;
3. opposed edge-policy implementations required by those evidence bytes; and
4. initial and counterpart-partial workspaces that fail visible and hidden
   executable checks, while each oracle passes both.

Applying a branch's correct first patch to its counterpart must fail that
counterpart. Pair-static digests replace only the designated discriminator and
must be identical within each dyad. Task IDs and host branch metadata are not
rendered into prompts. Receipts serialize the actual composed mapping from
task and query skin through evidence bytes to oracle first patch.

Evidence appears through three channels: public tests, documentation, and a
callsite. Bank, qualification, and transfer use disjoint path-name regimes.
Bank and qualification cross reference- and symbol-based search queries;
transfer uses only the held-out signature-query skin. Content digests are
unique within blocks and disjoint wherever the config declares a split
boundary.

## Primary event and statistical unit

The primary case event occurs before any generated verifier call. It requires:

- acquisition of the designated public evidence before the first changed
  patch;
- a first changed patch that passes that branch's visible and hidden checks;
  and
- that same patch failing when replayed on the counterfactual counterpart.

A dyad succeeds only if both branches satisfy the event. The dyad—not a branch,
trajectory, turn, or training row—is the inferential unit for acquisition.
Terminal workspace success, first-patch correctness, search efficiency,
verification, commit, and per-family/channel/query-skin rates are secondary
registered outcomes. Full sample-pool pass-if-any quantities are oracle
coverage diagnostics, never deployable selectors.

## Stage 0: model-free and immutable-design gates

Before a model call, smoke must prove:

- deterministic task regeneration, exact pair hygiene, opposed oracle replay,
  public/hidden separation, and content-disjoint blocks;
- path- and query-skin inventory plus a varying composed
  `task -> channel/path -> evidence -> first patch` map;
- strict first-tool parsing, canonical history, executable patch/test/submit,
  and rejection/failure scenario construction;
- exact transition inventory, within-dyad shuffled-target proof, zero think
  loss, equal rows/action mass, and transition-supercycle batching;
- tokenizer encoding and the registered logical batch/supercycle geometry;
- actual sampled/input/logical-token accounting and outcome-blind sample-prefix
  selection; and
- every later stage failing closed without the required ancestor receipt.

The exact start checkpoint is then audited against the apex on the registered
48 fresh non-coding contexts *before training*. It must already satisfy median
centered non-target logit drift at most 0.10 and mean entropy delta at least
−0.05. Failure is `LINEAGE_LOCALITY_INFEASIBLE`: it prevents training so
inherited drift cannot later be mislabeled as acquisition-update collateral.
Varentropy is diagnostic.

## Stage 1: outcome-free interface ladder

Run the exact start checkpoint on the fixed `interface_preflight` block at
answer rungs 1,024, 2,048, then 4,096, stopping at the first rung that passes.
Selection may use only invalid-action rate, every answer-limit/length contact,
runner identity, context fit, and whether the fixed injected search actually
exposes the designated public evidence. No correctness, hidden score, family
gain, or later block enters rung selection.

At the selected rung, invalid actions and all answer-limit contacts must each
be at most 0.05. No passing rung yields `INSTRUMENT_FAIL` and seals
qualification and training. The selected allowance is immutable for every
later behavioral arm.

Raw outputs are retained, while later assistant history contains only native
thinking plus the first valid parsed tool call. This prevents answer run-on
from becoming an accidental next-turn instruction and is identical across all
arms.

## Stage 2: two-block acquisition qualification

Use only the unchanged start checkpoint. Qualification A (seed 93100) and B
(seed 93200) each contain four tasks per training family and remain separate.
Each block runs:

- unassisted inferred-contract acquisition;
- host-injected correct public search evidence, a reachability ceiling;
- deterministic matched-operator search whose output is asserted to exclude
  the designated evidence path and marker; and
- explicit-contract acquisition, where the issue already states the policy.

Every block must independently satisfy:

- unassisted evidence acquisition before first patch at most 0.25;
- unassisted paired pre-verifier success at most 0.40;
- injected paired pre-verifier success at least 0.80;
- injected minus unassisted paired success at least +0.30;
- injected minus nondiscriminating-search paired success at least +0.30;
- explicit first-patch correctness at least 0.85 and unnecessary evidence
  acquisition at most 0.25;
- at least three families with injected paired success at least 0.75;
- all test/doc/callsite channels present and each at least 0.67 injected paired
  success;
- both reference and symbol query skins present and each at least 0.75 injected
  paired success; and
- invalid actions and all answer-limit contacts at most 0.05.

The blocks are never pooled to rescue a miss. Stop labels are
`INSTRUMENT_FAIL`, `NO_INJECTED_REACHABILITY`, and
`NO_ACQUISITION_HEADROOM`. Only `ACQUISITION_QUALIFIED` authorizes training.

## Stage 3: banks and training

All arms start from the exact transaction-replay checkpoint. The frozen source
replay bank is
`large_artifacts/qwen35_4b_transaction_invariant_recovery_curriculum/bank/transaction_replay.jsonl`,
SHA-256
`9c196d1e7e49881bbf151e1575c98811bcdca66e6ef38858c34f60f1256b9315`.

Each arm has 48 task blocks: 24 selected prior complete-loop blocks and 24 new
procedural tasks. After deterministic padding, every one of these nine
conditional transitions has exactly 48 rows:

1. `start_to_inspect_source`;
2. `ambiguous_source_to_inspect_evidence`;
3. `evidence_to_policy_patch`;
4. `explicit_source_to_patch`;
5. `rejected_patch_to_changed_patch`;
6. `failed_test_to_diagnose`;
7. `diagnosis_to_changed_patch`;
8. `patch_ok_to_verify`; and
9. `passed_test_to_commit`.

Thus each arm has 432 rows. Weighted answer-action mass is exactly 16,000
tokens at every transition per epoch, and think loss is zero everywhere.
Optimizer steps consume complete nine-transition supercycles; counterfactual
siblings co-occur within logical microbatches where the frozen scheduling
contract requires it, while their physical model forwards remain serial.

The preregistered arms are:

- `evidence_binding` (primary): aligned inferred dyads plus complete replay;
- `explicit_redundant`: equal-dose tasks whose issue already states the edge
  policy, while acquisition is redundant; and
- `shuffled_binding`: byte-identical primary prompts with the
  evidence-to-patch action targets exchanged only within each dyad. The target
  multiset is preserved exactly.

All use rank 32, alpha 64, dropout 0.05, LR 2e-5, batch size 4, accumulation 9,
three epochs, maximum encoded length 4,096, and seed 53. The logical batch is
four, but every row executes as an unpadded physical batch-of-one forward;
four numerators are summed before the frozen logical-microbatch denominator.
The maximum/minimum serial-forward token ratio across arms may be at most
1.01×. Training and merge receipts must prove exact equality of planned and
completed optimizer steps, identity with the locked encode-only schedule,
action mass, start identity, adapter-module geometry, and final weight hashes.
`TRAINING_COMPUTE_MISMATCH` is terminal and seals behavior evaluation. The
primary arm is fixed in advance; a control may not replace it because it scores
better.

## Stage 4: post-training locality

Before any trained behavior is generated, compare the merged primary directly
to the apex on the same 48 contexts. Require median centered non-target logit
drift at most 0.10 and mean entropy delta at least −0.05. Also report primary
minus start incremental drift and both entropy/varentropy changes.

Failure is `LOCALITY_FAIL` and seals every trained behavioral block and
Menagerie. Control-arm locality is reported but cannot rescue the primary.

## Stage 5: trained-family calibration

On seed 93400, generate the start and both trained controls before exposing the
primary. Hard score bounds must make every margin attainable. The primary must:

- reach paired pre-verifier success at least 0.65;
- beat start by at least +0.15;
- beat `explicit_redundant` and `shuffled_binding` by at least +0.10 each;
- keep invalid-action and unusable-cap rates within +0.02 of start; and
- retain explicit-contract first-patch correctness at least 0.83.

An unreachable bar is `CALIBRATION_INFEASIBLE`; an observed miss is
`CALIBRATION_FAIL`. Either seals transfer.

## Stage 6: family/path/query-held-out transfer

Development uses seed 93500 and confirmation seed 93600, each with eight tasks
per held-out family. No threshold, prompt, model, family, path, query, or decode
change is permitted between them.

For each block, generate and freeze start, apex incumbent, both trained
controls, normal-start, and six shallow stochastic trajectories for both start
and incumbent before scoring the primary comparison. First prove every
candidate-independent margin is attainable. Only then open the primary deep
run, compute-match the frozen pools, generate the matched-operator search
control, and prove those remaining margins attainable before opening any later
candidate scenario. For every case, select the first
trajectory-index prefix that meets or exceeds the primary deep run's *actual*
sampled-token and logical-model-token costs. Prefix membership uses costs only,
never outcomes. The primary sampling comparator is the stronger paired score
of the start and incumbent dual-overmatched prefixes. The full six-trajectory
pools are labeled oracle-only.

Each transfer block independently requires primary:

- paired pre-verifier success at least +0.08 over start and +0.08 over apex;
- at least +0.05 over each trained control;
- at least +0.05 over matched-operator nondiscriminating search, and injected
  evidence paired success at least 0.75 on the shifted substrate;
- at least +0.05 over the stronger dual-overmatched sample-more comparator;
- paired dyad-bootstrap 95% lower bound versus start at least 0.0;
- normal-start terminal success at least +0.05 over start;
- nonnegative paired delta on both held-out families and no family regression
  below −0.10;
- valid changed-patch recovery after a rejected patch and changed-patch
  recovery after a failed test each at least 0.95,
  verification at least 0.90, and commit at least 0.90;
- invalid-action and unusable-cap rates within +0.02 of start;
- explicit first-patch correctness at least 0.83 and unnecessary evidence
  acquisition at most 0.30;
- all three evidence channels represented, with paired pre-verifier success at
  least 0.50 in each; and
- the held-out signature-query skin present with paired pre-verifier success at
  least 0.50; and
- no more than 2.0 mean non-source inspections before the first patch.

Run confirmation only after a complete development pass. Stop labels are
`TRANSFER_DEV_INFEASIBLE`, `TRANSFER_DEV_FAIL`,
`TRANSFER_CONFIRM_INFEASIBLE`, and `TRANSFER_CONFIRM_FAIL`.

## Stage 7: legacy loop retention

Evaluate start and primary on two independent old-family blocks: broad recovery
(`lease_cache`, `quorum_value`, `pattern_router`, `rate_buckets`, seed 94000)
and transaction transfer (`atomic_reservations`, `atomic_debits`,
`atomic_membership_moves`, `atomic_patchset`, seed 94100). Each has eight tasks
per family and both normal and controlled rejected/failed starts.

On each block, primary normal terminal success and recovery terminal success
must each be at least start −0.03; rejected and failed transitions must each be
at least 0.95; verification and commit must each be at least 0.90; invalid and
unusable-cap rates must be within +0.02 of start; and no family terminal rate
may regress more than 0.10. Failure is `RETENTION_FAIL` and seals Menagerie.

## Stage 8: uncertainty diagnostics

Entropy and varentropy are stratified across quartiles and semantic next-action
buckets (`search_discriminator`, `read_other`, `patch`, `other`) on already-open
states. They may diagnose where the policy changed and may motivate a new
experiment. They never choose examples, assign correctness, weight tokens,
select checkpoints, route evaluation, or relax a gate.

## Stage 9: Menagerie

Only a complete white-box pass, including the exact matched-compute training
receipt, authorizes fresh paired quick and medium CLI
events at seeds 71311 and 71312. Seed freshness must be rechecked against the
public registry immediately before invocation. Candidate and apex use the same
backend and decode; only aggregate receipts are retained.

The benchmark strategy gate requires at least one tier delta of +0.02 or more
and no tier regression worse than −0.03. Failure is `MENAGERIE_FAIL`. A frozen
seed already consumed by another experiment is `MENAGERIE_SEED_UNAVAILABLE`.
Once this experiment writes its pre-call reservation, a failure before the
authenticated append-only aggregate event is `MENAGERIE_INSTRUMENT_FAIL`; the
seed is never called again. Analyzer-only failure after a complete logged event
is resumable because it performs no public call. A complete pass is
`COUNTERFACTUAL_EVIDENCE_ACQUISITION_POSITIVE`.

## Interpretation and prohibited rescues

A full white-box positive supports installation of an ambiguity-triggered
public-evidence search-and-binding policy on these procedural coding families.
It does not establish general active search, verifier-free correctness,
internal uncertainty measurement, or a universal agentic capability unlock. A
Menagerie pass would add cross-instrument evidence, not erase this boundary.

Prohibited after design lock:

- lowering a gate, dropping a family/channel/query skin, changing the parent,
  selecting a better control as primary, or pooling dev with confirmation;
- changing answer allowance after the outcome-free interface ladder;
- choosing sample prefixes with success, hidden scores, or candidate outcomes;
- using entropy/varentropy as correctness, selection, or loss pressure;
- training on benchmark content or opening Menagerie before authorization; or
- appending a new substrate, dose, repair, or follow-up result to this
  result-bearing directory.

Every failure, negative control, stopped stage, and external artifact receipt
is preserved. Any rescue is a new intake and experiment.
