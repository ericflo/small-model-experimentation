## Status

This is the preregistered design record. The scientific result has not run in

## Research program fit

Round 1 established C52: near-parity single-position FTPO causes generic
think-flow collateral. The most informative follow-up is not another dose of
the same recipe, but a selector × update-direction test at the boundary C52
predicts: confident distributional outliers. A held-out repository agent makes
the program's coding/tool-use north star explicit before blackbox spend.

## Frozen method

The same real rows feed conventional pairwise FTPO (`demote`) and bounded
positive-only reinforcement (`uplift`). The parent label-shuffled rows feed an
otherwise identical uplift control. Base logits qualify only focused,
spiky-conflicted wrong turns; full formulas, thresholds, gates, and low-dose
scope are in [preregistration.md](preregistration.md). The adversarial review
and every accepted risk are in [design_review.md](design_review.md).

## Smoke evidence (not a scientific result)

- All six generated repository bugs fail visible and hidden tests before the
  oracle patch and pass both afterward; inputs are not mutated.
- Tool JSON parsing ignores JSON-like text inside `<think>`, patch paths cannot
  escape `src/`, and test files are read-only to the agent.
- The uplift objective has zero direct rejected-token gradient at reference;
  demotion moves chosen up and rejected down; uplift deactivates at +0.5.
- A two-row exact-logit GPU smoke loaded the pinned base successfully.
- The first one-line coding suite saturated and was discarded. The hardened
  semantic suite's six-task calibration showed 2/6 deep and 1/6 matched short-
  trajectory final-workspace passes; these tiny adaptive-smoke numbers are not
  eligible for later hypothesis claims.
