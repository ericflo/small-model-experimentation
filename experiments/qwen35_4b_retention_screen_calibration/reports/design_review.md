# Adversarial Design Review

An eval-only instrument-calibration cell: nothing trains, nothing merges,
nothing promotes, no benchmark seed exists. Three independent review lenses
(preregistration-versus-implementation, statistical correctness, fail-closed
integrity) ran with adversarial verification of every non-minor finding.

- ONE MAJOR finding confirmed, zero refuted, and it was corrected before any
  commit or model event: the draft derived `recommended_band` and
  `adjudication_protocol` from the pooled SD of retention-correct LEVELS,
  while every band the outputs will govern adjudicates a same-screen DELTA
  versus a parent — common screen-difficulty variance inflates the level SD
  but cancels exactly in same-screen deltas, and independent per-arm noise
  makes the delta SD ~√2 × the level SD, so the draft calibrated against the
  wrong noise process with an unknowable bias direction. The governing
  estimand is now `delta_sd_pooled` (pooled across-screen sample SD of the
  per-screen delta-vs-clean_parent series over the four non-parent arms,
  ddof=1, df=3 each); the level SD remains in the receipt descriptively.
  Preregistration, code, config, README, and tests were amended together; a
  regression test pins the cancellation case (levels wobbling with screen
  difficulty at a constant −5 delta must read band 5 / `single_screen`).
- The contract and fail-closed lenses returned no confirmed defects: the
  formulas, boundaries (inclusive ≤2 / ≤3.5), stability-interval definition,
  arm list, screen-major run order, and historical readings match between
  the frozen text, the code, and the tests; all code files and all five
  composite receipts/trees are hash-pinned fail-closed; `--check` recomputes
  byte-identically; the screens regenerated byte-identically after the
  amendment (only the receipt's code pins changed).
- 59 unit tests green; smoke green; seeds 88,022–88,025 grep-fresh; overlap
  receipts zero against all nine predecessor gates, all inherited corpora,
  and each other; no promotion logic exists and no seed can be opened.

**Verdict:** `PASS_EXPENSIVE_RUN`.
