# Mechanics implementation review

**Review date:** 2026-07-13

Three independent read-only audits reviewed the stable mechanics implementation
before preparation. They used no benchmark content, hidden/future task content,
model load, GPU work, or model request.

## Initial verdict

`BLOCK`

The adversarial passes found and demonstrated the following concrete defects:

1. Arbitrary request IDs, a cross-family request-ID collision, and a fresh
   stage-one/parent stage-two derived-seed collision could pass the initial
   same-stage overlap checks.
2. `identity.py`, which owns the namespace and lineage verifier, was not frozen
   to the construction commit. Resolving a supplied lock path before checking
   it also made final-component and directory-component symlinks ineffective.
3. A fully written bundle whose process died before the separate `GENERATED`
   receipt was stranded even though the frozen protocol requires runner-free
   recovery of intact generated bytes.
4. Canonical JSON bytes and duplicate keys were not rejected. Restart
   promotion did not re-fsync a fully written predecessor that might have been
   observed after a crash before its original file fsync completed.
5. An authentication receipt could coexist with a missing invocation and still
   reach runner construction. Regression coverage did not exercise the full
   16-state transaction matrix, all-nine-invocation offline recovery, fsync
   order/faults, corrupt `COMPLETE`, or the previous-completion hash chain.
6. A pending review did not itself machine-block preparation.

## Resolution

- Every row now has an exact schema and canonical namespace/family/task/
  candidate seed key, with `id == SHA256(canonical_seed_key)`. The four request
  families contain exactly 676 unique IDs and keys. Paired causal arms retain
  exact within-successor identity and order.
- Parent/fresh comparisons require zero task-ID, request-ID, canonical-key,
  stage-one, stage-two, any-stage, all-parent prompt-text, terminal prompt-text,
  and terminal rendered-token-ID intersections. Explicit mutations for the
  demonstrated bypasses fail.
- The exact 13-file parent lineage is reverified at the request read.
  `identity.py` and its test are construction-frozen and checked before local
  imports. Lexical lock paths reject symlinks in every component before
  containment/resolution.
- Transactions accept only `PENDING`, `STARTED+bundle`,
  `STARTED+bundle+GENERATED`, and the full `COMPLETE` prefix. Every other one of
  the 16 existence states is terminal. An intact canonical bundle may be
  re-durabilized and promoted to `GENERATED` without a runner; malformed,
  noncanonical, duplicate-key, or hash-invalid bytes never are.
- Durable writes are exclusive and ordered directory fsync, file fsync,
  directory fsync. Restart re-establishes file and directory durability after
  canonical/schema/hash validation and before promoting preflight, `STARTED`,
  bundle, `GENERATED`, `COMPLETE`, or authentication bytes.
- Raw inventory and any existing authentication receipt are rechecked under the
  run lock before runner construction. All nine generated invocations recover
  and complete under a test that makes `VLLMRunner` construction fatal.
- Preparation requires exactly one passing marker and exactly one explicit
  no-model/GPU marker in this non-symlink report; the report is then source-
  hashed into the preoutcome and implementation lock.

## Append-only V2 recovery review

The first clean lock attempt failed closed before lock creation because the
generated construction manifest was requested from pre-construction commit
`e43c701e` instead of published construction commit `9fc288eb`. It created no
raw directory, runner, model load, GPU initialization, model request, or sampled
output. The original published preoutcome remains immutable at SHA-256
`80647e830ccb90026b30b00ea674d22aa247eba925b4ebe38d6ddad8b49e0d0e`;
the deterministic incident receipt has SHA-256
`056fd507e83eedbc45648bcb73b4972faa61d0d164b5e57a472dfa667583c5aa`.

Three independent adversarial re-reviews initially blocked the V2 repair. They
found that a pending review could write the incident before failing, historical
incident validation would reject legitimate later lock/raw artifacts, the V1
payload table was not required to match exactly, and dangling final-component
symlinks could be replaced by frozen writes. The corrected implementation now
checks this unique V2 verdict before every preparation write, separates the
one-time pristine boundary from pure historical reconstruction, requires exact
V1 payload-table equality, and rejects dangling symlinks without replacement.

The split frozen-source mapping is now explicit and symmetric in schema V4:
only `manifest.json`, `mechanics_public.jsonl`, and `mechanics_audit.jsonl` map
to `9fc288eb`; all remaining frozen design/code files map to `e43c701e`. Both
publish and verification paths require the exact per-file map and both commit
ancestries. The implementation-critical inventory closes over both preoutcome
receipts, the incident, every prepared payload, and all implementation inputs.

Independent model-free reconstruction produced all 1,984 requests with the
exact V1 payload table and byte-identical request/control payloads, zero model
loads, zero model calls, and zero sampled outputs. The complete fresh suite
passed 108 tests and 96 subtests; the focused mechanics suite passed 67 tests
and 72 subtests.
