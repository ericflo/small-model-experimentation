## Prior Evidence

## Novelty Claim

This is the first universal-curriculum trial to separate same-parent successful-sibling repair on empirically live residual skills from replay-based retention on saturated or thin skills.

## Mechanism

Shortest correct siblings stay within the deployed parent’s support, while residual-only allocation concentrates the finite treatment dose where greedy errors exist. Active replay and an unchanged all-skill gate must preserve excluded skills. The explanation is false if the trained candidate fails to strictly beat both parent and replay overall and on execute/induct/probe without regressing retention skills.

## Control Plan

- Baseline: authenticated explicit `replay_after_close` parent.
- Mechanism-falsifying control: independent exact-exposure replay continuation from the same parent.
- Shift/robustness: fresh all-13-skill local gate, conditional all-family aggregate transfer, independent higher-tier confirmation.
- Hidden boundary: procedural truth grades samples but is excluded from sampling input; `benchmarks/` is unread and aggregate seed 78,142 is sealed.

## Evidence Output

- Program evidence: inherited-lineage, sibling collection, per-residual-skill availability, selected source, exact-exposure, paired training/deployment, and conditional promotion receipts.
- Shared synthesis: update only if the result changes the supported interface law.
- Reusable artifact: residual-only sibling input builder with explicit all-skill retention contract.
- Stop condition: fewer than four qualified sibling tasks in any residual skill stops; no resampling, quota borrowing, threshold change, or oracle fallback.
