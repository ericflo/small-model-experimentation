## Novelty Claim

Can a clean context-local donor state captured from an explicit candidate token
be installed behind a different valid source token after real native thought,
then affect both a randomized computed consequence and candidate-conditioned
continuation? This also tests whether the simpler visible hypothesis scaffold
is the actual useful method.

## Mechanism

Generate one shared 512-token native thought. Append a fixed candidate-anchor
scaffold inside `<think>`. Pick a valid source alias from task ID without gold.
For every other public alias, capture layer-4--8 donor state in the identical
context and replace all 24 recipient J coordinates with those clean values.
Every task uses a frozen balanced permutation from alias tokens to operation
meaning, so a successful randomized result-label consequence requires the
patched identity to be consumed through prompt-local semantics.

## Control Plan

- Literal target text: deployable hypothesis-enumeration baseline.
- Full donor activation: causal upper bound at the same token/context.
- Mean donor coordinates: isolates a common non-candidate shift.
- Frozen additive J: tests whether anchoring alone repairs the prior negative.
- Two independent exact live-bf16 non-J arms: paired norm with <=1% realized
  projection into the full 24-coordinate J span.
- Rotated wrong donor: must carry its own consequence, not merely disrupt source.
- Ordinary concept-logit-lens replacement: repeats the parent's inert control.
- Clean shared-prefix and fully independent sample-more pools: capability
  baselines at frozen sampled-token and total-forward-compute match points.
