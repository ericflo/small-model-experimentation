# Adversarial Design Review: Semantic-Anchor Coordinate Branching

Completed before implementation or any model call. Two independent read-only
reviewers examined the positive clamp, the valid replication, the failed native
additive branch, and this proposed bridge. Their objections are resolved below.

## Verdict

Proceed only as a staged mechanism test. The design can distinguish semantic
anchoring from arbitrary-token failure, but mapped-consequence transport is not
capability. Continuations remain forbidden unless the complete J consequence
gate and every live numeric control pass.

## 1. A `cloud` recipient is not neutral

`cloud` is a concept in the frozen lens and an invalid operation alias. It would
confound transport with an out-of-distribution recipient. Use a valid public
source alias chosen from task ID without gold. Every other alias is a target
donor. This restores the positive experiment's valid source-to-target design.

## 2. The scaffold is not wholly native thought

Correct. The first 512 tokens are native; the candidate interface is forced.
All reports call this scaffolded native thought, not spontaneous thought,
internal certainty, global workspace, or consciousness evidence.

## 3. Donor acquisition already supplies a cheaper text intervention

Literal target text is a mandatory arm and later capability comparator. Donor
capture compute is charged. J that merely recreates text is mechanism evidence;
J capability must beat text by 5pp or establish a frozen real compute advantage.

## 4. Immediate target alias is only token writing

Direct identity is diagnostic only. The primary endpoint applies the target's
prompt-local operation to `[3,-1,2,0]`, maps the distinct result through a fresh
task-randomized label table, and scores that label. Direct success cannot open
continuation if this consequence fails.

## 5. A random label table may still be a lexical lookup

The task also rotates alias-to-operation meaning. Passing requires two contextual
links and a computation: alias to published operation, operation to a distinct
one-step list, and list to a later randomized one-token label. Result-label
identity and logits never construct the intervention.

## 6. Fixed alias semantics could be memorized from parent experiments

Every task receives a frozen balanced permutation of aliases to operations,
published in its prompt. Correctness and consequence expectations follow the
task-local permutation. Parent fixed `cat=reverse` identity cannot pass broadly.

## 7. The full donor carries much more than J

It is explicitly an upper bound and site-validity prerequisite. Text pass plus
full-donor failure is `NO_NATIVE_ANCHOR_STATE_TRANSPORT`, not evidence against a
particular J basis.

## 8. Additive J is not a generic control

It is an alternative mechanism. The exact alpha-one centered branches from the
prior negative are fixed without a sweep. If they pass at the anchor, record
`ADDITIVE_ANCHOR_TRANSPORT` and retain them as a capability rival; do not force a
donor-replacement narrative.

## 9. Wrong donor must prove identity, not disruption

A task-frozen derangement supplies a distinct donor. It must select its own
computed result label on >=50% while selecting the registered target on <=15%.
Source disruption alone never passes.

## 10. One non-J control can be lucky

Use two independent controls per task, donor, probe, and layer. Each matches the
paired realized donor-J norm after bf16 within `1e-5` and has <=1% projection
into the complete 24-coordinate J span. Report the worse outcome control.

## 11. Float orthogonality is not live orthogonality

The parent footgun is load-bearing. Candidate search, iterative correction, and
bounded lattice repair operate on the live bf16 residual. They see geometry
only. One failed numeric row blocks outcome retention and mechanics.

## 12. Desired donor state could recursively include earlier patches

Every layer target comes from the clean donor trajectory captured before any
patch. Repeated-patch activations never define later desired values. Tests mutate
patched state and require the clean desired tensors to remain byte-identical.

## 13. The hook could patch a suffix or repeat

All aliases/result labels must be one leading-space token. Source and donors
must have identical length and absolute anchor position. The hook count is one
per layer, batch is one, padding absent, and mechanics uses `use_cache=False`.
Suffix-causal activation invariance must be <=`1e-3`.

## 14. Cache behavior could invalidate later continuation

Mechanics deliberately uses cache-free full recomputation to match the positive
replication. If it passes, a separate outcome-blind implementation receipt must
show cache-anchor versus full-prefill logits within 0.05 and exact prefix-cache
invariance before any generated continuation.

## 15. Four tasks do not provide 44 independent statistical units

Correct. The task is the statistical unit for capability and uncertainty. The
44 source-target mechanics rows are a hard write/control gate, not a population
effect estimate. Qualification and confirmation bootstrap tasks only.

## 16. Mechanics breadth can hide dead concepts or labels

J consequence successes must span at least ten target aliases, ten randomized
labels, and all four tasks. Full transition matrices are reported. A few easy
tokens cannot open continuation.

## 17. The parameterized operations are underspecified

The diagnostic explicitly fixes `k=2`; its 12 one-step outputs are preregistered
and distinct. The diagnostic is separate from the hidden task pipeline and uses
no correctness field.

## 18. Native prefixes can close early or differ by arm

Candidate identity is appended only after one shared prefix reaches exactly 512
tokens. A natural close/EOS fails the task row rather than triggering a new
seed. All mechanics arms consume the identical prefix token IDs.

## 19. Mean donor state can masquerade as candidate control

Record an all-24 mean-donor coordinate arm. In capability, the target bank must
beat this common-shift baseline. Candidate-specific wrong-donor and breadth gates
provide complementary evidence.

## 20. A logit direction could explain an apparent J effect

Retain ordinary concept-logit-lens replacement, which was inert in the positive
replication. It is diagnostic, not a substitute for non-J or wrong-donor
controls.

## 21. Fresh procedural tasks may collide with ancestors

Fingerprint complete visible+hidden behavior plus task-local alias mapping.
Reject any overlap with all readable ancestor experiment artifacts, regenerate
before model load, and commit the manifest. Never read benchmark contents.

## 22. Branch instructions could reward blind adherence

The continuation says to investigate, reject if inconsistent, and revise. A
branch is not required to finish with its supplied target. Mechanics measures a
controlled diagnostic consequence; capability grades the frozen ensemble rule,
not supplied-target obedience.

## 23. Enumerating every first alias makes oracle coverage cheap

Coverage is secondary and insufficient. The deployable endpoint is one frozen
ensemble prediction. A later implementation should also record an unenumerated
derived task consequence when feasible; no coverage-only or diversity-only
result can pass.

## 24. J receives shared-prefix and donor computation for free

It does not. Resource receipts count prompt prefill, 512 native tokens, every
clean donor capture, source processing, scaffold token, cache/full audit,
continuation, close, and final slot. Sample-more match points are frozen before
labels at sampled-token and total logical model-token parity, plus full K=12.

## 25. Same seeds do not make backends comparable

Every arm and sample-more trace uses the same pinned Transformers bf16 SDPA
backend. The vLLM scaffold was removed because this measurement needs internals.

## 26. A mechanics positive could be called installed capability

Weights never change. Mechanics establishes a writable semantic register only.
A qualification/confirmation win over text and matched-compute sampling would
be test-time elicitation. Installation would require a later training/controller
experiment with its own clean matched baselines.

## 27. Failed mechanics can be rescued by neutral tokens, alpha, or layers

No. Valid-source recipient, all-24 donor values, layers 4--8, and fixed additive
alpha one are final. A failed explicit-anchor donor clamp retires native J
branching. Neutral-recipient, amplitude, token, and layer searches are new ideas
and are not authorized.

## Mandatory pre-GPU boundary

- complete fresh-data collision and prompt-leak audit;
- lens hash/rank/concept/token/layer audit;
- all 12 operation consequences proven distinct;
- task-balanced alias/operation and result-label permutations;
- source/donor position and length tests;
- clean-donor immutability, hook-count, suffix-causality, and patcher tests;
- exact two-control float geometry tests;
- task-label mutation proving mechanics outputs unchanged;
- model-free smoke with all model stages fail-closed; and
- pushed hashes for design, implementation, tests, lens, and public data before
  outcome-blind live-bf16 smoke.
