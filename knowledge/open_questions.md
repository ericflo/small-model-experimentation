# Open Questions

## Selection And Evidence

- What deployable evidence most reliably separates hidden-correct from hidden-wrong visible-pass candidates?
- Can generated counterexamples be useful without requiring hidden expected answers?
- When should selectors abstain instead of committing a plausible candidate?
- How much independent implementation consensus is needed before precision improves?

## Structured Compilers

- Which parts of structural latent compiler success come from typed slots, state supervision, curriculum, or Qwen features?
- Do high-performing compiler results survive more seeds, longer programs, and harder operator distributions?
- Can typed bytecode and latent slot approaches be unified without losing interpretability?

## Retrieval And Memory

- When does retrieval provide missing algorithmic hints versus distracting the model?
- Can verified memory help if it is used to construct tests or constraints instead of prompt context?
- What metadata should a skill/operator memory store to make retrieval selection reliable?

## Active Learning

- Which visible-only uncertainty signals predict that an added example will help?
- Are acquisition gains family-specific, especially for numeric and date/time transformations?
- Can active evidence policies transfer across substrates?

## Repository Process

- Which experiment-level metadata should become hand-curated rather than generated?
- Which repeated local utilities deserve promotion into a shared package?
- What minimum result schema would make cross-experiment metric comparison easier without burdening fast pilots?
