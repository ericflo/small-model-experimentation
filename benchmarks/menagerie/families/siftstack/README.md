# siftstack

## Capability Axis

Information triage over a noisy fictional ledger: resolve aliases, ignore
near-miss distractors, apply later-record supersession, and aggregate a count,
sum, or current/latest value from document-bound facts.

## Task Description

`siftstack` is a single-turn registry-ledger task family. Each item presents a
short ledger made of generated entries about fictional entities. Relevant facts
are interleaved with distractor entities, near-miss names, unrelated fields, and
older values that may be superseded later in the same document. The model must
answer from the document alone and end with a final line:

`ANSWER: <value>`

The generator first creates structured fact records, then renders the visible
ledger from those exact records. The oracle and scorer compute ground truth from
the same record list, after alias closure and supersession, so the rendered
document and answer state cannot diverge.

Core semantics:

- Alias declarations are global and retroactive. If `A, also filed as B` appears
  anywhere in the ledger, every record under either name belongs to the same
  entity. Alias chains are resolved transitively.
- Facts are keyed by `(resolved entity, field, slot)`. A later fact with the
  same resolved entity, field, and slot replaces the earlier value for that key.
- Distractor facts for other entities, unrelated fields, and near-miss names
  never count unless an alias declaration explicitly joins them to the target.
- The query states the aggregation rule in full, including alias use and
  latest-record replacement.

## Document Grammar

All task vocabulary that denotes entities, fields, slots, and status values is
generated from a seeded fictional syllable stock. Tokens are assembled from
nonword fragments, require at least two fragments per generated token, and avoid
embedded common-word forms. Entity display names use one or two title-case
tokens; fields, slots, and status codewords use lowercase generated tokens.
Numbers are two- or three-digit base-10 integers.

Atom-mode ledger lines use short English scaffolding around generated tokens:

- Fact: `<entry>. <entity> filed <field>/<slot> = <value>.`
- Alias: `<entry>. <entity>, also filed as <alias>.`
- Update: `<entry>. <entity> revised <field>/<slot> = <value>.`
- Distractor: the same fact/update forms, but for an unrelated entity, unrelated
  field, or near-miss name.

Episode mode remains single-turn with `max_turns == 1`, but uses a denser
compressed surface to stay within the 800-character observation limit:

- `A <entity>=<alias>` for aliases.
- `F <entity> <field>/<slot>=<value>` for facts.
- `U <entity> <field>/<slot>=<value>` for replacing updates.
- `Q ...` for the fully specified query.

Near-miss names are generated by changing one syllable, swapping a second token,
or reusing one visible alias token without an alias declaration.

## Query Scheme

Each item asks exactly one query type, selected deterministically from the seed,
level, mode, and item index:

- `COUNT`: after merging aliases and keeping only the last value per
  entity-field-slot key, count how many current target slots in the requested
  field family satisfy the stated value or status condition.
- `SUM`: after the same alias and supersession pass, sum the current numeric
  values across all target slots in the requested field family.
- `LATEST`: after the same alias and supersession pass, report the current value
  for one requested target field/slot.

The query always names the target entity and says to include all filed aliases.
It also states that later entries replace earlier entries for the same filed
entity, field, and slot.

## Scoring

Scoring is binary exact match after canonical normalization. The scorer extracts
the last line matching `ANSWER:` case-insensitively, strips surrounding
whitespace, lowercases codeword answers, and compares numeric answers as
base-10 integers, so leading zeros are ignored for numeric golds. Units,
commas, explanations on the answer line, and non-integer numeric forms fail.
There is no partial credit.

## Novelty Statement

The three nearest public benchmarks are:

- [DROP](https://arxiv.org/abs/1903.00161): a reading-comprehension benchmark
  requiring discrete operations such as addition and counting over natural
  paragraphs. `siftstack` differs by using procedurally generated fictional
  ledgers, explicit alias closure, explicit latest-record supersession, and
  binary exact-match scoring instead of natural answer spans/F1-style metrics.
- [RULER](https://arxiv.org/abs/2404.06654): a synthetic long-context benchmark
  with needle retrieval, multi-hop tracing, and aggregation tasks. `siftstack`
  differs by staying short and char-bounded, using registry records rather than
  haystack text or word-frequency extraction, and requiring alias resolution plus
  overwrite semantics before aggregation.
- [bAbI](https://arxiv.org/abs/1502.05698): synthetic QA tasks for prerequisite
  reasoning skills such as fact chaining, induction, and deduction. `siftstack`
  differs by avoiding story-world templates, using fresh fictional registry
  vocabulary, scoring only a final canonical `ANSWER:` value, and combining
  aliases, near-miss decoys, supersession, and count/sum/latest aggregation in
  the same controlled ledger.

## Difficulty Ladder

- L1: short document, no aliases, no updates, one or two distractor entities,
  and a single-fact count or latest query. A weak model should score well above
  zero.
- L2: one alias pair, more distractors, and count or sum queries across mentions
  of the aliased entity.
- L3: alias chain depth two, supersession updates, near-miss distractor names,
  and queries that require resolving aliases before applying updates.
- L4: frontier setting with alias depth at least two, multiple interleaved
  updates, near-miss names, and aggregation requiring alias resolution and
  supersession together. Atom prompts approach but never exceed 1200 characters.

Episode mode is a single-turn composited variant. Regardless of level, each
episode item includes aliases, supersession, and a near-miss decoy using the
compressed grammar, with the observation capped at 800 characters.

## Anti-Degeneracy

Generated batches deterministically mix `COUNT`, `SUM`, and `LATEST` queries.
Count golds are distributed over 2 through 9. Sum golds are widened by varying
slot counts and two- or three-digit values. Latest-value golds alternate between
two- or three-digit integers and fictional status codewords. During batch
generation, answer histograms are tracked and deterministic resampling or range
widening prevents any single gold answer from occupying more than about ten
percent of non-tiny batches. Near-miss decoys are assigned plausible but
different aggregates from the target whenever possible.
